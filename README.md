## 设计背景

rktask 源自对传统 RTOS 的反思。

早期嵌入式项目中，我们长期使用**裸机前后台系统**——主循环轮询函数指针链表。这种模式在功能简单时够用，但一旦任务数超过 4 个，问题接踵而至：长任务阻塞全局、任务间通信靠全局变量加轮询、所有任务共享调用栈导致局部变量污染。

转向 FreeRTOS 后，虽然解决了上述问题，但引入了新的困扰：**15 个优先级数字该选哪个？优先级翻转是什么？为什么不设置互斥锁会导致系统崩溃？** 对于大多数小型 MCU 项目（任务 ≤ 16、功能固定），这些概念带来的认知负担远超其实际价值。

rktask 的答案是：回到问题的本质——在设计理念中，嵌入式系统只有两种任务，不需要优先级数字。

## 三项黄金准则

| 准则 | 目标 | 验证方式 |
|:----|:----|:---------|
| **最高性能** | PendSV 切换 ≤ 9 条指令 (M3+) | 反汇编直接计数 |
| **最小体积** | 内核 Flash ≤ 1.5KB, RAM ≤ 200B | 链接器 map 文件 |
| **最简单** | 全部 API ≤ 15 个 | 头文件函数声明计数 |

这三个数字不是口号——**每行代码都要对它们负责**。

- 9 条 PendSV 指令意味着一次上下文切换约 36 周期（72MHz 下约 500ns），即使 1ms tick 下频繁切换，开销也可以忽略
- 1.5KB Flash 的约束逼迫每字节都有明确用途——绝不包含"可能有用"的可选特性
- 14 个 API、两种任务类型，没有优先级数字、没有优先级翻转、没有优先级继承——用户 10 分钟就能掌握全部概念

## 核心设计决策

### 两类型任务：COOP + PREEMPT

传统 RTOS 有 0~N 个优先级数字。rktask 认为大部分嵌入式场景只需要区分两类工作：

```
COOP    协作式 — 不可被抢占，执行完整原子操作
        适合：协议解析、状态机、Flash 编程、命令处理

PREEMPT 抢占式 — 时间片轮转，可被自动剥夺
        适合：周期性采集、通信轮询、软件定时器、看门狗喂狗
```

**COOP 优先调度**——只要有 COOP 任务就绪，PREEMPT 任务完全得不到 CPU。这强制开发者将关键逻辑放在 COOP 中，把不重要的后台活动放在 PREEMPT 中，从根本上消除了"低优先级饿死"问题。

### COOP 不需要锁

COOP 任务运行期间不会被任何任务打断，因此 COOP 操作共享资源不需要互斥锁。这一特性大幅降低了并发编程的心智负担——**不必考虑"什么地方可能被抢断"**。

### BASEPRI 临界区 (M3+)

rktask 选择 BASEPRI 而非 PRIMASK。BASEPRI 只屏蔽优先级低于某阈值的中断，临界区内：

- **SysTick 仍可触发**（延时精度不受临界区影响）
- **高优先级外设中断仍可响应**（UART、TIM、DMA 不会丢数据）
- **临界区嵌套天然支持**（BASEPRI 值在栈中保存/恢复）

### Delta 编码延时队列

每个 SysTick（周期由 `RK_TICK_MS` 配置，默认 1ms），只递减延时队列头节点（O(1)），而非遍历全部延时任务（O(N)）。把复杂度从高频路径搬到了低频路径。

## 关键指标

| 指标 | M3/M4/M7 | M0+ |
|:----|:--------:|:---:|
| PendSV 切换指令 | **9** | 13 |
| 内核 Flash | **~1.2 KB** | ~1.3 KB |
| 内核 RAM | **~200 B** | ~200 B |
| 每任务 TCB | **32 B** | 32 B |
| SysTick 延时处理 | **O(1)** （Delta 编码） | 同左 |
| rk_task_self() 时间 | **O(1)** （内置 ID） | 同左 |

## 架构支持

| 架构 | 芯片示例 | 临界区 | PendSV |
|:----|:---------|:-------|:-------|
| **Cortex-M0+** | STM32G0 / STM32L0 | PRIMASK | 13 条指令 |
| **Cortex-M3** | STM32F103 | BASEPRI | 9 条指令 |
| **Cortex-M4** | STM32F407 | BASEPRI | 9 条指令 |
| **Cortex-M7** | STM32H743 | BASEPRI | 9 条指令 |

编译期自动检测，零配置。

## 核心功能一览

| 功能 | 说明 |
|:----|:-----|
| **任务调度** | COOP 协作式 + PREEMPT 时间片轮转双类型调度 |
| **事件标志** | 32 位事件组，AND/OR 等待模式，ISR 可安全发布 |
| **Delta 延时** | 任务延时 `rk_task_delay(ms)`，O(1)/tick |
| **栈保护** | 栈底金丝雀检测 `rk_task_stk_check()` |
| **临界区** | M0+ PRIMASK / M3+ BASEPRI 自适应 |
| **调试断言** | `RK_ASSERT()` 条件编译 |

内核共 **14 个 API**，全部以 `rk_task_` 为前缀。
